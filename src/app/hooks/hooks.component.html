NgOnChanges

Purpose: ngOnChanges is designed to handle changes to the component's input properties.

Parent to Child Data Transfer: It is used when data is passed from a parent component to a child component using the @Input decorator.

Child Component Setup:

Define Method: Implement the ngOnChanges method in the child component to respond to changes in input properties.

Import Required Modules: Import OnChanges and SimpleChanges from Angular core in the child component.

Class Implementation: Add OnChanges to the class declaration after OnInit.

Note: It's only working for strings, not for arrays and objects.

Example Simplified:

Define Input Properties: Use the @Input decorator in the child component to receive data from the parent.

Implement ngOnChanges: Create the ngOnChanges method in the child component to handle changes in the input properties.

NgOnInit

Purpose: ngOnInit is designed to perform component initialization logic.

Timing: It runs after Angular first displays the data-bound properties.

Setup in Child Component:

Import OnInit: Import OnInit from Angular core in the child component.

Add OnInit to Class: Add OnInit to the class declaration after @Component.

Implement ngOnInit Method: Define the ngOnInit method in the class.

Call Sequence and Usage:

Call Sequence: ngOnInit is called once after the first ngOnChanges.

Usage: Suitable for initializing data or fetching data from a service.

NgDoCheck

Purpose: ngDoCheck is a lifecycle hook in Angular for implementing custom change detection.

Custom Change Detection: It allows you to perform custom logic to detect and respond to changes in component state that Angular's default change detection might miss.

Setup in Child Component:

Import DoCheck: Import DoCheck from Angular core in the child component.

Add DoCheck to Class: Add DoCheck to the class declaration after @Component.

Implement ngDoCheck Method: Define the ngDoCheck method in the class to implement your custom change detection logic.

Call Sequence and Usage:

Call Sequence: ngDoCheck is called during every change detection cycle.

Usage:ngOnChanges is only for get changes of strings but ngDoCheck is for all to check the changes.

NgOnDestroy

Purpose: Perform cleanup logic before the component is destroyed.

Timing: Runs just before Angular destroys the component.

Setup in Child Component:

Import OnDestroy: Import OnDestroy from Angular core.

Add to Class: Add OnDestroy to the class declaration.

Implement Method: Define the ngOnDestroy method.

Call Sequence and Usage:

Call Sequence: Called once before the component is removed from the DOM.

Usage:It is used to destroy the component (Which mean hide the entire component)
